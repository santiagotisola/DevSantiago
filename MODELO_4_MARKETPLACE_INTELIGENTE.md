## 🔍 **Dores Reais dos Moradores**

### Pain Point 1: Desorganização
```
❌ PROBLEMA:
"Não sei que parceiros/produtos tem disponível no meu condomínio"
"Academia tem? Farmácia tem? Qual é a mais perto?"
"Preciso clicar em 5 lugares diferentes pra achar ofertas"

✅ SOLUÇÃO MODELO 4:
Dashboard único com TODOS os parceiros e produtos
Feed inteligente: "Para você hoje"
Recomendações: "Você comprou Whey, quer BCAA?"
```

### Pain Point 2: Preços Confusos
```
❌ PROBLEMA:
"Qual é o preço final? Preço + desconto + frete = ?"
"Aquela academia de 6 meses atrás tava mais barata?"
"Preciso calcular na mão"

✅ SOLUÇÃO MODELO 4:
Preço final transparente em CARDS grandes
Histórico de preços (mostra se tá mais caro)
Alerta: "Produto caiu 10% de preço!"
```

### Pain Point 3: Desconfiança
```
❌ PROBLEMA:
"Como sei se esse parceiro é bom?"
"Já comprei lá? Qual foi minha experiência?"
"Ninguém avaliza o parceiro"

✅ SOLUÇÃO MODELO 4:
Rating ⭐⭐⭐⭐⭐ de moradores
Avaliações com foto e comentário
Badge "✅ Verificado" (admin aprova)
Seu histórico com parceiro (5 compras, 10 devoluções)
```

### Pain Point 4: Comunicação Ruim
```
❌ PROBLEMA:
"Como falo com o parceiro?"
"Ele respondeu no WhatsApp mas cadê o registro?"
"Não tenho whatsapp do Restaurante"

✅ SOLUÇÃO MODELO 4:
Chat integrado no app (histórico completo)
Bot que responde: "Qual seu horário?"
Video call direto
Notificações em tempo real
```

### Pain Point 5: Falta de Controle
```
❌ PROBLEMA:
"Pedi um produto ontem, e agora?"
"Quando chega?"
"Como rastreio?"

✅ SOLUÇÃO MODELO 4:
Status em tempo real: "PENDENTE → RESPONDIDO → ACEITO → ENVIADO → ENTREGUE"
Localização do entregador (se parceiro usar app)
Aviso de entrega 15min antes
Confirmação de recebimento
```

### Pain Point 6: Perda de Histórico
```
❌ PROBLEMA:
"Qual o preço da Academia 6 meses atrás?"
"Já comprei esse produto?"
"Paguei quanto?"

✅ SOLUÇÃO MODELO 4:
Histórico completo de compras
Recibos digitais em PDF
Gráfico: "Você gastou R$ 5.000 em Academia esse ano"
Comparar preços no histórico
```

### Pain Point 7: Oportunidades Perdidas
```
❌ PROBLEMA:
"Não vejo ofertas novas"
"Perdi desconto que expirou"
"Não sabia que tinha essa promoção"

✅ SOLUÇÃO MODELO 4:
Notificações inteligentes:
  "Academia está com BLACK FRIDAY - 30% off"
  "Seu Whey favorito em estoque!"
  "Comprou Proteína? Desconto em Creatina hoje"
Favoritos (salva produtos interessantes)
```

### Pain Point 8: Impossibilidade de Bulk
```
❌ PROBLEMA:
"Quero comprar Whey pra todo bloco (20 pessoas)"
"Precisa de desconto, mas pra quantidade grande"
"Ninguém organiza isso"

✅ SOLUÇÃO MODELO 4:
Compra coletiva (monta um grupo)
Parceiro vê demanda agregada
Desconto por volume automático
Rastreamento por participante
```

## 🗄️ **Tabelas do Banco (Modelo 4)**

```
-- CORE MARKETPLACE
marketplace_partners (1 registro = múltiplos condominiums)
├─ id, name, logo, rating, isVerified
├─ visibility_mode, createdByCondominiumId

condominium_marketplace_partners (junção N:N)
├─ partnerId, condominiumId, branchName, isActive

marketplace_products
├─ id, partnerId, condominiumId
├─ name, description, price, discount, shippingCost
├─ imageUrl, stock, category, rating

-- REQUISIÇÕES & COMUNICAÇÃO
marketplace_product_requests
├─ id, productId, residentId, partnerId
├─ quantity, notes, status, createdAt

marketplace_chat_messages (chat em tempo real)
├─ id, requestId, senderId, message, timestamp

-- INTELIGÊNCIA & FIDELIDADE
resident_favorites (salvar produtos/parceiros)
├─ residentId, productId, savedAt

resident_purchase_history (histórico)
├─ residentId, productId, quantity, price, date

loyalty_points
├─ residentId, points, tier, lastUpdated

product_reviews
├─ productId, residentId, rating, comment, images

-- ANALYTICS & BUSINESS
partner_sales_metrics (vendas por período)
├─ partnerId, condominiumId, totalSales, productsSold

marketplace_notifications (log de notificações)
├─ id, residentId, type, message, sent_at

marketplace_complaints (para moderação)
├─ id, partnerId, description, status
```
